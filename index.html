After downloading, extract the contents of the ZIP file to a location on your computer.
Here's how you can do it on both Windows and Ubuntu:



Windows:

Using File Explorer:


  • Navigate to the location where you downloaded the ZIP file.
  • Right-click on the ZIP file.
  • Select "Extract All" from the context menu.
  • Choose the destination folder where you want to extract the files.
  • Click "Extract."


Using Third-Party Software:

  • If you have software like WinZip or WinRAR installed, you can right-click the ZIP file.
  • Choose the option to extract the files using the respective software.
  • Follow the prompts to select the extraction location and complete the process.


Ubuntu (or other Linux distributions):

Using Archive Manager (GUI):


  • Navigate to the location where you downloaded the ZIP file using your file manager.
  • Right-click on the ZIP file.
  • Select "Extract Here" to extract the files to the current directory.
  • Alternatively, choose "Extract to..." to specify a different extraction location.


Using Command Line (Terminal):

  • Open a terminal window.
  • Navigate to the directory where the ZIP file is located using the cd command.
  • Use the unzip command followed by the name of the ZIP file to extract its contents.
  • Optionally, you can specify a destination directory for extraction:

unzip filename.zip -d /path/to/destination

Once extracted, you can run the Yikes YTD application by double-clicking the executable file.
Following these steps will extract the contents of the ZIP file, allowing you to access and use the Yikes YTD application on your computer.

Contribution Guidelines

Before making contributions, please review our guidelines below.

  • Follow the existing code style and formatting conventions used in the project.

  • Use meaningful variable names and comments to improve code readability.

  • Ensure that your code is well-documented, especially for complex functions or modules.

  • Create a new branch for each feature or bug fix you are working on.

  • Use descriptive branch names that reflect the purpose of the changes.

  • Write clear and concise commit messages that describe the changes being made.

  • Use imperative mood (e.g., "Fix typo in README" instead of "Fixed typo in README").

  • Reference relevant issues or pull requests in your commit messages using hashtags (e.g., "#123").

  • Write unit tests and/or integration tests for new features or bug fixes.

  • Ensure that all existing tests pass before submitting your changes.

  • Add new tests to cover any added functionality or edge cases.

  • Update the project documentation (README, API docs, etc.) to reflect any changes or new features.

  • Include instructions for setting up and running the project locally, if applicable.

  • Provide examples or usage scenarios to help users understand the changes.

  • Submit pull requests from your forked repository to the main repository.

  • Include a clear description of the changes introduced by the pull request.

  • Ensure that all contributed code adheres to the project's licensing terms.

  • If you are contributing code that you did not write yourself, ensure that it is compatible with the project's license.

  • Participate in code reviews for other contributors' pull requests.

  • Provide constructive feedback and suggestions for improvement.

  • Reach out to the project maintainer(s) if you have any questions or concerns about the contribution process.

  • Communicate any significant changes or proposed features before starting work on them.

  • Adhere to the project's code of conduct and treat all contributors with respect and professionalism.

  • Report any instances of harassment, abuse, or misconduct to the project maintainers.

    • Contribution Process
